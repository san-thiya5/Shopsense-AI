# SQL Schema — ShopSense AI

## Overview
This document defines the complete database structure for ShopSense AI using Firebase Firestore — a real-time NoSQL cloud database.

---

## Database: Firebase Firestore

### Collection 1: inventory

| Field | Data Type | Description |
|-------|-----------|-------------|
| id | String (Auto) | Unique product ID generated by Firebase |
| name | String | Name of the product (e.g., Sugar, Rice) |
| quantity | Number | Current stock quantity |
| unit | String | Unit of measurement (kg, litre, piece, packet, box) |
| price | Number | Price per unit in Rupees |

---

### Collection 2: sales

| Field | Data Type | Description |
|-------|-----------|-------------|
| id | String (Auto) | Unique sale ID generated by Firebase |
| product | String | Name of the product sold |
| quantity | Number | Quantity sold |
| unit | String | Unit of the quantity sold |
| price | Number | Price per unit at time of sale |
| total | Number | Total amount = quantity x price |
| date | String | Date of the sale |
| time | String | Time of the sale |

---

### Collection 3: udhaar

| Field | Data Type | Description |
|-------|-----------|-------------|
| id | String (Auto) | Unique udhaar ID generated by Firebase |
| customer | String | Name of the customer |
| item | String | Item taken on credit |
| amount | Number | Amount owed by the customer |
| date | String | Date of the credit entry |
| status | String | Payment status — Paid or Unpaid |

---

## Why Firebase Firestore?

- Free tier (Spark Plan) — no cost for small projects
- Real-time updates — data reflects instantly across all devices
- No SQL knowledge needed — simple key-value document storage
- Works perfectly with React via Firebase SDK
- Hosted on Google Cloud — reliable and fast for Indian users (Mumbai server)

---

## Tools Used

- Firebase Firestore (NoSQL Database)
- Firebase SDK v9
- React.js for frontend integration
